# 实验三：类型检查

## 实验描述

在生成AST后，对程序进行类型检查。__TypeCheck.h__ 和 __TypeCheck.cpp__ 是本次实验需要编辑的文件，里面已经定义了一些数据结构和函数，给出了类型检查的整体框架。其中，入口是 __check_Prog__ 函数。你可以根据需要，额外定义自己的函数或数据结构。

### 输入格式

一棵 AST，数据结构定义在 TeaplaAst.h。你在 __check_Prog__ 函数中拿到的即是整个程序的 AST 的 root。

注意，相比实验二，本次作业的语法中新增了 void 和 bool 两个类型 token。

### 输出格式

如果出现了类型检查错误，请输出错误的位置和原因。__TypeCheck.cpp__ 里提供了一个 __error_print__ 函数，你可以使用它输出错误信息然后退出程序。

### 测试样例

测试样例在 tests/ 文件中，其中每个文件的末尾都以注释的形式提供了参考输出。你输出的错误信息不必和参考输出字字相同。

### Getting Started

环境

```
gcc 11.4.0
bison 3.8.2
flex 2.6.4
```
如果你在实验二中已经搭建好了环境，那么理应可以直接运行。

运行

在本目录下运行

```
make
```

即可编译所有文件，并对所有 testcase 进行测试。在你开始编写本作业的代码之前，可以运行 make 测试你的环境是否可以运行。
